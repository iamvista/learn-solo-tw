# Prompt 模板：專案規劃

## 1. 專案構思 Prompt

```
我想做一個 [描述你的想法]。

目標受眾是 [誰會用這個東西]。
我希望它能解決的問題是 [問題描述]。

請幫我：
1. 評估這個想法的可行性
2. 列出需要的核心功能（按優先順序）
3. 建議 MVP（最小可行產品）應該包含哪些功能
4. 預估用 Vibe Coding 方式開發的時間
```

## 2. 需求拆解 Prompt

```
我要做一個 [專案名稱]，一句話描述：[描述]。

請幫我用以下格式拆解需求：

## 使用者故事
- 身為 [角色]，我想要 [功能]，以便 [目的]

## 頁面結構
列出所有需要的頁面和每頁的功能

## 資料需求
這個專案需要儲存哪些資料？

## 第三方服務
需要串接什麼外部服務？

## 技術建議
用什麼技術棧最適合這個專案？
```

## 3. 使用者流程 Prompt

```
我正在做 [專案名稱]，核心功能是 [描述]。

請幫我畫出完整的使用者流程：
1. 使用者第一次進入網站會看到什麼？
2. 他們最可能做的第一個動作是什麼？
3. 從開始到完成核心任務的每個步驟是什麼？
4. 每個步驟可能遇到的問題和解決方式

請用箭頭（→）表示流程方向。
```

## 4. 競品分析 Prompt

```
我想做 [你的專案描述]。

請幫我分析市場上類似的產品：
1. 列出 3-5 個類似產品/網站
2. 每個的優點和缺點
3. 我的專案可以怎麼差異化？
4. 有哪些功能是「必須有」的（因為用戶已經習慣了）？
```

## 5. 功能優先順序 Prompt

```
以下是我的專案的功能清單：
[列出所有功能]

請幫我用 MoSCoW 方法分類：
- **Must Have（必須有）**：沒有就不算完成
- **Should Have（應該有）**：很重要但不是第一版必須
- **Could Have（可以有）**：有的話加分
- **Won't Have（暫不做）**：未來再考慮

並說明你的判斷理由。
```
