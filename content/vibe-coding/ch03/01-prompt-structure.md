# 3-1 Prompt 的基本結構與原則

## 學習目標

- 掌握 Prompt 的四層結構框架
- 理解「好 Prompt」的五大原則
- 能夠獨立寫出結構清晰的 Prompt

## 內容大綱

### 1. 為什麼 Prompt 這麼重要？（3 分鐘）
- Prompt 就是你跟 AI 的「溝通介面」
- 同樣的工具，不同的 Prompt，結果天差地別
- 實際對比：同一個需求，三種不同的 Prompt 品質

### 2. CRAFT 框架（10 分鐘）

#### C — Context（上下文）
- 你是誰？你在做什麼？
- 這個專案的背景是什麼？
- 範例：「我是一個賣手工皂的創業者，想做一個銷售頁面」

#### R — Role（角色）
- 讓 AI 扮演特定角色
- 範例：「你是一位資深前端工程師」
- 什麼時候需要設定角色、什麼時候不用

#### A — Action（行動）
- 明確的動詞指令
- 「幫我建立」vs.「幫我分析」vs.「幫我修改」
- 一次一個行動，不要混在一起

#### F — Format（格式）
- 你要表格？列表？程式碼？
- 指定語言、風格、長度
- 範例：「用 Markdown 表格整理」

#### T — Tone（語調）
- 正式 / 輕鬆 / 技術性 / 口語化
- 適用於文案類輸出

### 3. 五大原則（5 分鐘）
1. **具體優於模糊** — 「藍色按鈕，圓角 8px」>「好看的按鈕」
2. **分步優於一次** — 大任務拆成小步驟
3. **範例優於描述** — 給 AI 看你要的樣子
4. **限制優於放任** — 告訴它「不要做什麼」
5. **迭代優於完美** — 先求有，再求好

### 4. 實戰演練（5 分鐘）
- 練習：把一個模糊的需求改寫成 CRAFT 格式
- Before / After 對比
- 學員跟著做：改寫自己的 Prompt
