# 3-3 除錯與迭代：當 AI 搞錯的時候

## 學習目標

- 建立正確的除錯心態：AI 出錯是常態
- 學會識別三種常見錯誤類型
- 掌握有效的修正溝通技巧

## 內容大綱

### 1. AI 一定會犯錯（3 分鐘）
- 不是工具壞了，是溝通需要調整
- 犯錯率跟你的 Prompt 品質成反比
- 心態：每次修正都在讓結果更好

### 2. 三種錯誤類型（8 分鐘）

#### 類型一：做錯了（功能不對）
- 症狀：按鈕按了沒反應、顯示的資料不對
- 原因：你的需求描述有歧義
- 修正法：更具體地描述期望行為
- 範例對話演示

#### 類型二：做醜了（視覺不對）
- 症狀：排版混亂、顏色醜、間距奇怪
- 原因：你沒給視覺參考
- 修正法：截圖標註 + 具體指出哪裡要改
- 範例對話演示

#### 類型三：做壞了（程式報錯）
- 症狀：紅色錯誤訊息、頁面空白
- 原因：程式碼有 bug（AI 也會寫 bug）
- 修正法：把錯誤訊息整段貼給 AI
- **關鍵技巧**：錯誤訊息就是最好的 Prompt

### 3. 萬用除錯 Prompt（7 分鐘）
- 模板：「我遇到了這個問題：[貼錯誤訊息]。我期望的行為是 [描述期望]。目前的行為是 [描述現狀]。請幫我修正。」
- 進階：附上相關程式碼的截圖
- 最高效的做法：在 Cursor 中直接用 ⌘+K，AI 會自動看到上下文

### 4. 迭代的藝術（5 分鐘）
- 第一輪：讓它先做出基本功能（60 分）
- 第二輪：修正錯誤和明顯問題（80 分）
- 第三輪：微調細節和視覺（90 分）
- 第四輪：最終打磨（95 分）
- 知道什麼時候「夠好了」

### 5. 實戰：修一個壞掉的頁面（5 分鐘）
- Vista 故意做一個有 bug 的頁面
- 帶學員一步步用 AI 修好它
- 強調過程和溝通方式
