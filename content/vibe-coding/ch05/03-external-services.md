# 5-3 串接外部服務：表單、Email、資料庫

## 學習目標

- 理解「前端」和「後端」的基本概念
- 學會串接 3 種常見的外部服務
- 讓你的作品從「展示」升級到「可營運」

## 內容大綱

### 1. 為什麼需要外部服務？（3 分鐘）
- 到目前為止：你做的是「靜態網站」
- 靜態 = 不會記住東西、不會寄信、不會存資料
- 外部服務 = 給你的網站加上「記憶」和「能力」
- 好消息：不用自己架伺服器，用別人的服務就好

### 2. 服務一：進階表單處理（8 分鐘）
- 上一堂用的是嵌入式表單，現在學自訂表單
- Formspree：用 HTML form + action 屬性
- 實作：自訂漂亮的聯絡表單
  - 姓名、Email、訊息
  - 表單驗證（必填、格式檢查）
  - 送出後的感謝頁面
- 進階：用 Webhook 串接 Notion / Google Sheets

### 3. 服務二：自動寄信（8 分鐘）
- 場景：用戶填完表單後自動收到一封感謝信
- EmailJS：前端直接寄信的服務（免費額度）
- 實作步驟：
  - 註冊 EmailJS
  - 建立 Email 模板
  - 在網頁中加入 EmailJS SDK
  - 用 AI 寫串接程式碼
- 測試：填表 → 收到信

### 4. 服務三：簡易資料庫（8 分鐘）
- 場景：想存使用者的計算結果 / 測驗分數
- Supabase：免費的雲端資料庫（像 Firebase 但用 PostgreSQL）
- 為什麼推薦 Supabase？
  - 免費額度慷慨
  - 介面直覺
  - AI 很熟悉它
- 實作：讓診斷測驗的結果存到 Supabase
  - 建立 Supabase 專案
  - 建立資料表
  - 用 AI 寫資料寫入程式碼
  - 在 Supabase Dashboard 看到資料

### 5. 安全注意事項（3 分鐘）
- API Key 不要放在前端程式碼裡（為什麼？怎麼做？）
- 環境變數的概念
- 用 Supabase Row Level Security（RLS）
- 基本原則：前端可以被看到，敏感的東西放後端
