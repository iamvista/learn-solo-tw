# 4-2 設計：用 AI 生成頁面佈局

## 學習目標

- 用 AI 產出網站的視覺設計
- 學會從參考網站「借鑑」設計元素
- 產出可以直接進入開發的 HTML/CSS 原型

## 內容大綱

### 1. 設計思維入門（3 分鐘）
- 你不需要會 Figma、不需要會畫圖
- AI 時代的設計流程：描述 → 生成 → 調整
- 「好設計」的三要素：一致性、呼吸感、對比

### 2. 參考網站分析（5 分鐘）
- 怎麼「看」一個網站的設計？
  - 顏色（主色、輔色、背景色）
  - 字體（標題、內文、大小層級）
  - 間距（留白的感覺）
  - 佈局（一欄、兩欄、網格）
- 實作：分析學員選的參考網站
- 用 Chrome DevTools 看顏色和字體（超簡單示範）

### 3. 用 AI 生成首頁設計（10 分鐘）
- Prompt 範例：
  ```
  請幫我設計一個個人品牌官網的首頁，使用 HTML + Tailwind CSS。

  視覺風格：參考 [URL]，乾淨簡約，深色導覽列 + 白色內容區
  主色：#2563eb（藍色），輔色：#1e293b（深灰）

  頁面區塊：
  1. 導覽列（LOGO + 選單連結）
  2. Hero 區塊（大標題 + 副標題 + CTA 按鈕）
  3. 服務項目（3 欄卡片）
  4. 關於我（左圖右文）
  5. 頁腳（版權資訊 + 社群連結）

  手機版要自適應。
  ```
- 看 AI 怎麼回應
- 即時預覽效果

### 4. 修正與調整（8 分鐘）
- 顏色不喜歡？「把主色改成 #059669（綠色）」
- 間距太擠？「每個區塊之間的間距加大到 80px」
- 想換佈局？「Hero 區塊改成左文右圖」
- 加入動畫？「Hero 標題加入淡入效果」
- 每次修正都即時預覽

### 5. 產出所有頁面的設計（5 分鐘）
- 首頁設計確認後，用同樣的風格延伸到其他頁面
- Prompt 技巧：「參考 @index.html 的設計風格，幫我建立 about.html」
- 確保所有頁面視覺一致

## 課後作業（必做）

- [ ] 完成首頁的 AI 設計（至少迭代 3 輪）
- [ ] 完成其他所有頁面的設計
- [ ] 在手機上預覽，確認行動版顯示正常
